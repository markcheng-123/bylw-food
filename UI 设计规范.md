# UI 设计规范（毕设优秀标准）

项目名称：基于 SpringBoot 和 Vue 的本地特色美食论坛的设计与实现  
版本：v1.0  
日期：2026-03-18  

## 1. 设计目标

1. 视觉统一：前台用户端与后台管理端均保持统一的设计语言与组件规范。
2. 体验流畅：关键流程（登录、搜索、发布、审核）交互顺手，反馈明确。
3. 内容突出：以图文内容为核心，强化“美食”“本地”“分享”氛围。
4. 响应式适配：桌面端与移动端均可稳定使用，满足答辩演示与论文截图需求。

## 2. 视觉方向（推荐）

定位：生活化内容社区 + 本地特色美食信息平台。  
气质关键词：温暖、干净、有食欲、有秩序、可信。  

### 2.1 色彩策略

推荐一套“暖橙主色 + 米白底色 + 深灰文字”的组合，适合美食氛围且不俗。

- 主色（Brand）：暖橙，用于主按钮、强调标签、选中态
- 辅色（Accent）：红棕或番茄红，用于热度、推荐等点缀（谨慎使用）
- 背景（Background）：米白/奶油白，减少纯白刺眼
- 中性（Neutral）：深灰/浅灰，用于文字与分割
- 状态色（Status）：成功绿、警告橙、错误红、信息蓝

落地要求：

- 全站颜色必须从变量表取值，不允许页面中随意出现新色值。
- 前台与后台可以在同一色板基础上做“饱和度/对比度”区分：前台更温暖，后台更克制。

### 2.2 字体与层级

原则：可读性优先，层级清晰，避免同一页面字号混乱。

- H1 页面标题：28-36px
- H2 模块标题：20-24px
- H3 卡片标题：16-18px
- 正文：14-16px
- 辅助信息：12-14px

落地要求：

- 标题与正文必须区分字重与颜色层级（例如标题更深、正文稍浅）。
- 行高：正文建议 1.6 左右，长文内容不拥挤。

### 2.3 网格与间距（Spacing System）

建议使用 4px 或 8px 作为基础单位，统一间距系统。

- 页面外边距：桌面端 64-96px；平板 24-32px；移动端 16px
- 模块间距：24/32/48px（按页面密度选择）
- 卡片内边距：16/20/24px

落地要求：

- 页面栅格统一（例如 12 栅格），列表与详情页对齐。
- 禁止“看着差不多”式间距，必须在设计稿/代码中落为固定值。

### 2.4 圆角、阴影与质感

- 圆角：卡片 14-18px；按钮 10-14px；输入框 10-12px
- 阴影：低对比柔和阴影，悬浮时略增强

落地要求：

- 阴影与边框不要同时过强，避免“廉价感”。
- 卡片 hover 状态必须有轻微抬升（transform）+ 阴影变化，提升精致度。

## 3. 响应式规范

至少覆盖四档断点：

- 1440px：宽屏桌面
- 1200px：常见桌面
- 768px：平板
- 390px：移动端

落地要求：

- 所有列表页必须在 390px 下可读，筛选区需折叠为抽屉/弹层（推荐）。
- 详情页在移动端应改为纵向结构，侧边推荐区下移。

## 4. 组件规范（必须统一）

### 4.1 按钮 Button

层级：

- Primary：主按钮（提交/发布/通过）
- Secondary：次按钮（取消/返回/次要操作）
- Ghost/Text：文本按钮（轻量操作）

状态：

- 默认、Hover、Active、Disabled、Loading

落地要求：

- 不同页面的主按钮颜色、圆角、尺寸必须一致。
- 危险操作使用 Danger 样式，并配二次确认（后台尤其需要）。

### 4.2 输入 Input / Textarea

规范：

- 聚焦态边框色统一（使用 Brand 色的浅版本）
- 错误态具备错误文案与红色提示
- 文本域字符统计（发布页建议）

### 4.3 标签 Tag / Badge

用途：

- 分类标签、状态标签（待审核/已发布/驳回）、热度标签

落地要求：

- 状态标签必须与状态色统一绑定，避免不同页面同状态不同颜色。

### 4.4 卡片 Card

适用：

- 列表页、美食推荐、攻略推荐、商家展示

规范：

- 统一封面比例（例如 16:10 或 4:3）
- 标题最多两行，超出省略
- 摘要最多两行，超出省略
- hover 抬升 2-4px

### 4.5 列表与分页 List/Pagination

规范：

- 列表为空时展示空状态组件（图标+文案+引导按钮）
- 分页位置统一（列表底部居中或右对齐，二选一）

### 4.6 弹窗与提示 Modal/Toast

规范：

- 重要操作（删除、下架、驳回）必须二次确认
- 提示文案明确“发生了什么/下一步怎么办”

### 4.7 上传 Upload（发布页与入驻页关键）

规范：

- 支持拖拽（可选），必须支持点击选择与预览
- 显示上传进度/成功/失败
- 失败提示包含原因（类型不允许/大小超限/网络失败）

## 5. 页面模板规范

### 5.1 前台首页

目标：首屏有氛围、有记忆点，适合答辩开场。

模块建议：

- 顶部导航
- Banner 主视觉（城市+美食氛围图或暖色渐变）
- 分类入口（图标或小卡片）
- 热门推荐（卡片）
- 最新发布（瀑布流或网格）
- 优质攻略（横向卡片）
- 商家推荐（可选）

验收点：

- 首屏信息不拥挤，主视觉清晰
- 图片风格统一，不出现清晰度差、比例不一致

### 5.2 美食列表页

结构：

- 搜索栏（固定在首屏）
- 筛选条（分类/地区/排序）
- 卡片列表 + 分页

移动端策略：

- 筛选条折叠为“筛选”按钮，打开抽屉设置条件

### 5.3 美食详情页

结构（桌面端）：

- 左侧主内容：封面图、标题、标签、正文
- 右侧信息栏：店铺信息、操作区（点赞）、推荐内容
- 底部评论区

验收点：

- 内容排版舒适（段落、行高、图片间距合理）
- 评论区层次清晰（头像、昵称、时间、内容区分）

### 5.4 发布页

结构：

- 分组卡片：基础信息/店铺信息/图片上传/内容描述
- 底部提交区：保存草稿（可选）/提交审核

验收点：

- 错误提示不打断用户，定位明确
- 上传区有缩略图网格预览

### 5.5 用户中心

结构：

- 资料卡（头像、昵称、简介、统计数字）
- Tab：我的发布/我的评论/我的点赞

验收点：

- 体现“社区个人主页感”，避免像纯后台

### 5.6 后台管理

目标：强调秩序与效率。

结构：

- 左侧菜单 + 右侧内容区
- 顶部面包屑（可选）
- 表格统一样式 + 状态标签 + 操作列固定（建议）

验收点：

- 状态色与标签统一
- 审核通过/驳回按钮位置统一
- 驳回必须弹窗填写原因

## 6. 交互与动效规范

建议动效原则：少而精，服务信息反馈与质感提升。

- 页面切换：轻微淡入（150-250ms）
- 卡片 hover：抬升 + 阴影增强（120-180ms）
- 提交动作：Loading 状态 + 成功提示，避免用户误以为没生效

## 7. 可用性与可访问性（加分项）

- 表单控件具备清晰的 label 与错误提示
- 对比度满足可读性（浅色背景上的文字不能过浅）
- 键盘可操作基本流程（至少登录、发布可 Tab 切换）

## 8. UI 验收清单（最终交付必过）

1. 色板、字号、间距、圆角、阴影统一。
2. 关键组件（按钮/输入/卡片/标签/弹窗/上传/表格）统一。
3. 空/错/加载状态齐全。
4. 390px 移动端无错位、无遮挡、无横向滚动。
5. 首页、列表、详情、发布、后台页面有明确视觉亮点，适合答辩展示。
