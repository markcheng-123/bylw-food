# 需求规格说明书（SRS）

项目名称：基于 SpringBoot 和 Vue 的本地特色美食论坛的设计与实现  
版本：v1.0  
日期：2026-03-18  

## 1. 引言

### 1.1 编写目的

本文档用于将开题报告中的研究目标、需求分析与功能模块，转化为可开发、可测试、可验收的需求规格说明。本文档将作为后续原型设计、UI 设计、接口设计、数据库设计、开发实现与毕业设计验收的统一依据。

### 1.2 项目背景

本地特色美食信息目前主要分散在短视频平台、社交媒体与综合点评网站中，信息来源零散、筛选成本高、真实性与参考价值难以保障；同时商业推广占比高、普通用户参与度低，优质内容难以沉淀。为此，需要构建一个面向本地特色美食的内容发布、交流互动与规范化管理的平台。

### 1.3 目标与范围（严格对照开题报告）

系统目标：

1. 提供本地特色美食信息的集中发布与展示。
2. 提供互动机制（评论、点赞）促进经验交流。
3. 提供分类、关键词检索等信息组织能力，提高获取效率。
4. 提供美食攻略展示与推荐，提升平台内容可读性与实用性。
5. 引入商家入驻与后台内容审核机制，提高内容规范性与可信度。

范围边界：

- 本系统为 Web 应用，采用前后端分离架构。
- 前端：Vue；后端：SpringBoot（Java）；数据库：MySQL；接口：RESTful API。
- 本系统不以电商交易为目标，不包含在线支付与配送等能力（除非后续论文扩展明确提出）。

### 1.4 名词与缩略语

- SRS：Software Requirements Specification，需求规格说明书
- RESTful API：基于 HTTP 的资源风格接口
- UI：用户界面（视觉与交互）
- RBAC：基于角色的权限控制（本项目可简化为“角色+接口鉴权”）

## 2. 总体描述

### 2.1 系统角色

- 普通用户：注册登录、浏览、搜索、发布美食内容、上传图片、评论、点赞、查看攻略、维护个人资料。
- 商家用户：提交入驻资料，查看审核状态；审核通过后可在平台展示商家信息（展示方式由产品设计确定）。
- 管理员：用户管理、内容审核与管理、分类管理、商家入驻管理、攻略管理（如启用）。

### 2.2 系统运行环境

- 客户端：现代浏览器（Chrome/Edge），支持移动端浏览。
- 服务端：SpringBoot 服务运行环境（JDK 版本以项目配置为准）。
- 数据库：MySQL（建议 8.x），字符集 `utf8mb4`。

### 2.3 系统约束

- 技术栈约束：必须使用 Vue + SpringBoot + MySQL + RESTful API。
- 数据与内容约束：内容发布需具备审核/状态控制，避免违规内容直接公开（至少具备“待审核/已发布/驳回/下架”状态）。
- 视觉与交互约束：UI 需统一风格、布局合理、交互顺畅，满足毕设优秀展示标准。

## 3. 业务流程概述

### 3.1 注册与登录流程

1. 用户提交注册信息（账号/密码/昵称等）。
2. 系统校验合法性并创建用户。
3. 用户登录获取会话凭证（Token/Session）。
4. 登录态用于后续发布与互动。

### 3.2 美食内容发布流程

1. 登录用户进入发布页填写信息（名称、分类、店铺信息、描述、图片等）。
2. 系统进行参数校验与图片上传校验。
3. 内容进入“待审核”（默认）或“直接发布”（视毕设规则与权限而定）。
4. 管理员审核通过后对外展示。

### 3.3 浏览检索流程

1. 用户在列表页浏览内容，支持按分类筛选、关键词搜索、排序。
2. 点击进入详情页查看图文与互动数据。

### 3.4 评论点赞流程

1. 登录用户在详情页发表评论。
2. 登录用户对美食内容点赞（需防重复）。
3. 系统实时返回结果并更新展示数据。

### 3.5 商家入驻与审核流程

1. 商家提交入驻资料（名称、地址、联系方式、简介、图片等）。
2. 管理员审核通过/驳回并记录原因。
3. 审核通过后商家信息可展示在前台（入口与展示方式在 UI 规范中定义）。

## 4. 功能需求（FR）

本章节按模块列出功能需求，供开发与测试逐条对照。

### 4.1 用户端：认证与账号

FR-U-01 用户注册  
用户可通过账号注册访问系统；系统需校验账号唯一性、密码强度（至少长度限制）与必填项。

FR-U-02 用户登录  
用户可通过账号密码登录；登录成功后获得可用于后续接口访问的凭证；登录失败给出明确提示。

FR-U-03 退出登录  
用户可安全退出，客户端清理登录态。

### 4.2 用户端：个人信息管理

FR-U-04 查看个人资料  
用户可查看昵称、头像、简介、联系方式等资料。

FR-U-05 编辑个人资料  
用户可修改昵称、简介、联系方式；支持头像上传（限制文件类型与大小）。

### 4.3 用户端：美食内容发布

FR-C-01 发布美食内容  
用户可发布美食内容，字段至少包括：标题/名称、分类、店铺信息（可选或必填由产品定义）、文字描述、图片（至少 1 张，最多 N 张）。

FR-C-02 图片上传与预览  
发布页支持图片上传与预览；上传失败需提示原因；支持删除重传。

FR-C-03 内容状态控制  
内容应具备状态：草稿（可选）、待审核、已发布、驳回、下架；前台列表仅展示“已发布”。

### 4.4 用户端：浏览与查询

FR-B-01 美食列表展示  
支持分页展示；卡片显示封面、标题、分类、点赞数、发布时间（字段可按 UI 设计调整）。

FR-B-02 分类筛选  
按分类筛选美食内容。

FR-B-03 关键词检索  
支持按标题/关键字搜索，返回匹配结果。

FR-B-04 排序  
支持按“最新发布/最热（点赞数或热度）”排序（至少实现一种）。

FR-B-05 详情页展示  
详情页展示完整图文、分类标签、店铺信息、互动区与评论区。

### 4.5 用户端：互动交流

FR-I-01 评论发布  
登录用户可发表评论；评论长度限制；支持删除自己评论（可选）。

FR-I-02 点赞  
登录用户可点赞/取消点赞；同一用户对同一内容不可重复点赞。

FR-I-03 互动数据展示  
详情页展示评论数量、点赞数量；列表页展示点赞数等摘要。

### 4.6 用户端：美食攻略展示

FR-S-01 攻略列表  
用户可浏览攻略列表，支持分页与基础筛选（可选）。

FR-S-02 攻略详情  
展示攻略图文内容；可关联推荐美食内容（可选）。

### 4.7 商家入驻

FR-M-01 入驻申请提交  
商家可提交入驻申请，字段至少包括：商家名称、地址、联系方式、简介、图片材料。

FR-M-02 查看审核状态  
申请人可查看当前状态：待审核/通过/驳回，并能看到驳回原因（如驳回）。

### 4.8 后台管理：用户管理

FR-AU-01 用户列表  
管理员可查看用户列表，支持搜索与分页。

FR-AU-02 用户状态管理  
管理员可禁用/启用用户（可选）；禁用用户无法登录或无法发布（规则需明确）。

### 4.9 后台管理：内容审核与管理

FR-AC-01 待审核列表  
管理员可查看待审核内容列表，支持按分类/时间筛选。

FR-AC-02 审核操作  
管理员可通过/驳回；驳回需填写原因；审核结果写入审核记录。

FR-AC-03 下架与恢复  
管理员可将已发布内容下架（可选），下架后前台不可见。

### 4.10 后台管理：分类管理

FR-CA-01 分类 CRUD  
管理员可新增/修改/删除分类（删除需校验是否被引用，或采用禁用方式）。

### 4.11 后台管理：商家入驻管理

FR-AM-01 入驻审核列表  
管理员可查看商家入驻申请列表。

FR-AM-02 审核通过/驳回  
管理员可通过/驳回，驳回需原因，支持查看申请详情。

### 4.12 后台管理：攻略管理（如启用）

FR-AS-01 攻略 CRUD  
管理员可新增/编辑/发布/下架攻略。

## 5. 非功能需求（NFR）

### 5.1 易用性与交互

- NFR-UX-01：关键流程（注册、登录、发布、搜索、审核）步骤清晰，错误提示具体。
- NFR-UX-02：页面需具备加载态、空状态、错误态，不允许“白屏无提示”。

### 5.2 UI 视觉质量（毕设优秀标准）

- NFR-UI-01：全站统一色板、字号体系、圆角与阴影体系。
- NFR-UI-02：卡片、按钮、输入框、标签、弹窗、表格等组件视觉统一。
- NFR-UI-03：响应式适配至少覆盖 1440/1200/768/390 四档宽度。
- NFR-UI-04：关键页面（首页、列表、详情、发布、后台）具备明显视觉亮点与一致风格。

### 5.3 性能

- NFR-P-01：列表页分页加载，避免一次性加载大量数据。
- NFR-P-02：图片需压缩/限制尺寸，建议支持懒加载（前端实现）。

### 5.4 安全与合规

- NFR-S-01：密码不明文存储（哈希）。
- NFR-S-02：接口鉴权，未登录不可发布/点赞/评论/后台操作。
- NFR-S-03：上传文件校验类型与大小，避免任意文件上传。
- NFR-S-04：内容输出进行 XSS 风险控制（至少对富文本/用户输入进行转义或过滤）。

### 5.5 可维护性

- NFR-M-01：后端分层清晰，接口返回统一格式，异常统一处理。
- NFR-M-02：接口文档与实际实现保持一致，可用 Apifox 维护。

## 8. 验收标准（Checklist）

功能验收：

1. 用户注册登录正常，鉴权生效。
2. 美食内容可发布、上传图片、进入待审核并可审核发布。
3. 前台可列表展示、分类筛选、关键词搜索、进入详情页。
4. 评论点赞可用且防重复。
5. 后台用户管理、内容审核、分类管理、商家审核功能可用。
6. 攻略展示可用（如实现）。

UI 验收：

1. 关键页面风格统一，排版规范。
2. 响应式适配完成，无明显错位与横向滚动。
3. 空/错/加载状态齐全，交互反馈明确。

工程验收：

1. 数据库脚本可复现，初始化数据完善。
2. 接口文档齐全，前后端联调记录可追踪。
