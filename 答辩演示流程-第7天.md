# 答辩演示流程（第7天版本）

## 1. 演示目标

系统可稳定演示，主要流程可走通，界面风格统一，具备社区交互和后台管理闭环。

## 2. 账号准备

- 普通用户账号：用于登录、发布、评论、点赞、商家入驻申请
- 管理员账号：用于后台审核内容、分类管理、商家审核

## 3. 演示顺序（建议）

1. 登录系统（普通用户）
2. 进入首页，展示热门/最新内容和分类入口
3. 进入美食列表和详情，演示筛选、查看详情
4. 在详情页演示点赞和评论
5. 进入发布页提交一条新内容
6. 进入攻略列表与攻略详情页
7. 进入个人中心，展示我的发布/评论/点赞
8. 进入商家入驻页，提交申请
9. 退出普通用户，登录管理员
10. 进入后台管理，演示内容审核、分类管理、商家审核
11. 切回普通用户，验证审核结果可见

## 4. 讲解要点

- 架构：Spring Boot + Vue + MySQL
- 权限：普通用户与管理员分权
- 社区化：评论、点赞、个人中心数据联动
- 管理闭环：前台提交 + 后台审核 + 状态回流

## 5. 风险预案

- 若端口冲突：先关闭占用端口的旧进程再启动
- 若数据太少：提前准备演示数据（发布几条内容、攻略和申请）
- 若网络波动：尽量本机本地演示，减少外网依赖


前端：
