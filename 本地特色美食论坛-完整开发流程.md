# 本地特色美食论坛完整开发文档（落地版）

项目：`bylw-food`  
技术栈：SpringBoot 2.7 + MyBatis-Plus + MySQL + Vue3 + Element Plus + Vite  
文档版本：v2.3  
更新日期：2026-03-25

## 1. 项目目标

建设一个面向 C 端用户、商家与管理员的本地美食论坛系统，覆盖：

- C 端：浏览、搜索、发布、点赞评论、个人中心
- 商家端：入驻申请、资质认证、厨师管理、商家菜品发布
- 管理端：内容审核、商家审核、分类管理、数据概览、地图监控
- AI：全局悬浮式美食顾问（SSE 流式）

## 2. 系统架构

### 2.1 前端

- 框架：Vue3 + TypeScript + Vue Router + Pinia
- UI：Element Plus + 自定义暖色主题
- 目录：`frontend/src/views`、`frontend/src/components`、`frontend/src/api`

### 2.2 后端

- 框架：SpringBoot + MyBatis-Plus
- 分层：`controller -> service -> mapper -> entity`
- 鉴权：JWT + `AuthInterceptor`
- 目录：`backend/src/main/java/com/bylw/foodforum`

### 2.3 数据层

- MySQL 库：`food_forum`
- 12 张核心业务表（见《数据库设计说明书》）

### 2.4 文件上传

- 本地上传目录：`app.upload-path`
- OSS 开关：`app.upload.oss-enabled`
- 可切换到阿里云 OSS

## 6. 关键业务规则

1. 帖子默认待审核（`status=0`），审核通过才前台可见。  
2. 点赞同用户同帖子唯一（`uk_like_user_post`）。  
3. 商家通过审核后，用户角色升级为商家（`role=2`）。  
4. 商家资质证件变更后进入复审状态（`merchant_profile.status=0`）。  
5. 首页统计口径是全站口径，不按当前用户过滤。


### 7.2 前端

```bash
cd frontend

npm run dev
```

默认端口：`5173`

### 7.3 生产构建校验

```bash
cd frontend && npm run build
cd backend && mvn -DskipTests compile
```

## 8. 本次系统梳理发现的纰漏（需修复）

### P0（高优先级）

1. 配置存在明文 AI Key 回退值，存在泄漏风险。  
文件：`backend/src/main/resources/application.yml`

### P1（中优先级）

1. 前端 AI 组件硬编码 `http://localhost:8080`，生产环境会失效。  
文件：`frontend/src/components/ai/AiAdvisorWidget.vue`
2. 前端 API 基础地址硬编码本地地址，建议环境变量化。  
文件：`frontend/src/api/http.ts`
3. 打包体积较大（`AdminMapMonitorView`、`echarts` chunk 过大），首屏性能可继续优化。

### P2（优化项）

1. 部分中文报错文案在源码中出现乱码痕迹，建议统一 UTF-8 清理。  
文件：`backend/src/main/java/com/bylw/foodforum/service/impl/*.java`
2. 菜品食材/过敏原采用逗号拼接，后续可拆子表以增强结构化检索。
