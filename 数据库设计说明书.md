## 3. 角色与状态字典

### 3.1 用户角色（`user.role`）

- `1`：普通用户
- `2`：商家用户
- `9`：管理员

### 3.2 用户状态（`user.status`）

- `1`：正常可登录
- 其他值：禁用/不可登录

### 3.3 内容状态（`food_post.status`）

- `0`：待审核
- `1`：已通过（前台可见）
- `3`：驳回

### 3.4 商家申请状态（`merchant_application.status`）

- `0`：待审核
- `1`：通过
- `2`：驳回

### 3.5 商家资料状态（`merchant_profile.status`）

- `0`：待审核/复审中
- `1`：通过
- `2`：驳回

### 3.6 评论状态（`comment.status`）

- `1`：正常
- 其他值：屏蔽/不可见

### 3.7 攻略状态（`strategy.status`）

- `0`：待审核/草稿
- `1`：可见
- `2/3`：下线或驳回（按实现策略）

## 5. 表结构（真实落地）

> 字段来源：`backend/src/main/resources/sql/init.sql` 与线上库 `information_schema`。

### 5.1 `user` 用户表

主键：`id`  
唯一约束：`uk_user_username(username)`

主要字段：

- `username` `varchar(64)` 用户名（唯一）
- `password` `varchar(255)` BCrypt 密文
- `nickname` `varchar(64)` 昵称
- `avatar` `varchar(255)` 头像地址
- `phone` `varchar(20)` 手机
- `email` `varchar(128)` 邮箱
- `role` `tinyint` 角色
- `status` `tinyint` 状态
- `created_at` / `updated_at`

索引：`idx_user_role`、`idx_user_status`

### 5.2 `food_category` 分类表

主键：`id`  
唯一约束：`uk_category_name(name)`

主要字段：

- `name` 分类名
- `icon` 小图标值
- `sort` 排序
- `status` 启停状态
- `description` 分类介绍

索引：`idx_category_status`、`idx_category_sort`

### 5.3 `food_post` 帖子主表

主键：`id`

主要字段：

- `author_user_id` 作者
- `category_id` 分类
- `title` 标题
- `summary` 摘要
- `content` 正文
- `cover_image` 封面图
- `address` 地址文本
- `per_capita` 人均
- `status` 审核状态
- `view_count` / `like_count` / `comment_count`
- `published_at` 审核通过时间
- `created_at` / `updated_at`

索引：`idx_post_author`、`idx_post_category`、`idx_post_status`、`idx_post_created`

### 5.4 `food_image` 帖子图片表

- `post_id` 帖子ID
- `image_url` 图片地址
- `sort` 排序
- `created_at`

索引：`idx_food_image_post`

### 5.5 `food_dish` 菜品表（商家增强）

- `post_id` 帖子ID
- `dish_name` 菜品名
- `ingredients` 食材（逗号拼接）
- `image_url` 菜品图（可多图拼接）
- `allergens` 过敏原（逗号拼接）
- `sort` 排序

索引：`idx_food_dish_post`

### 5.6 `comment` 评论表

- `post_id`
- `user_id`
- `parent_id`（预留回复链）
- `content`
- `status`
- `created_at` / `updated_at`

索引：`idx_comment_post`、`idx_comment_user`、`idx_comment_created`

### 5.7 `like_record` 点赞记录

- `post_id`
- `user_id`
- `created_at`

唯一约束：`uk_like_user_post(user_id, post_id)`（防止重复点赞）

索引：`idx_like_post`、`idx_like_user`

### 5.8 `merchant_application` 商家入驻申请

- `user_id`
- `merchant_name`
- `contact_name` / `contact_phone`
- `business_license`
- `address`
- `description`
- `status`
- `reject_reason`
- `submitted_at` / `audited_at`
- `created_at` / `updated_at`

索引：`idx_merchant_user`、`idx_merchant_status`

### 5.9 `merchant_profile` 商家正式资料

- `user_id`（唯一）
- `merchant_name` / `store_name`
- `store_address`
- `average_cost`
- `business_license_url`
- `food_safety_cert_url`
- `chef_team_intro`
- `status`
- `reject_reason`
- `last_audited_at`
- `created_at` / `updated_at`

唯一约束：`uk_merchant_profile_user(user_id)`

### 5.10 `chef_info` 厨师资料

- `merchant_profile_id`
- `chef_name`
- `title`
- `avatar_url`
- `intro`
- `sort`
- `status`
- `created_at` / `updated_at`

索引：`idx_chef_profile`

### 5.11 `strategy` 城市攻略

- `author_user_id`
- `title` / `summary` / `content`
- `cover_image`
- `status`
- `sort`
- `published_at`
- `created_at` / `updated_at`

索引：`idx_strategy_status`、`idx_strategy_created`

### 5.12 `audit_record` 审核日志

- `business_type`
- `business_id`
- `operator_admin_id`
- `result`
- `remark`
- `created_at`

索引：`idx_audit_business(business_type,business_id)`、`idx_audit_operator`

## 6. 关键统计口径（必须统一）

### 6.1 首页统计 `/api/home/stats`

- `totalPosts`：全站已通过帖子数（`food_post.status = 1`）
- `todayNewPosts`：今日新增且已通过帖子数（`status = 1 and created_at in today`）

说明：这是“全站口径”，不是当前登录用户口径。

### 6.2 热度计算

- `heatScore = view_count + like_count + comment_count`

用于热门排序与首页热门推荐。

## 7. 事务与一致性规则

- 点赞：`like_record` 先去重，再同步 `food_post.like_count`
- 评论：评论写入后刷新 `food_post.comment_count`
- 审核：审核结果写业务表状态，并记录 `audit_record`
- 商家通过后：写入或更新 `merchant_profile`，并将用户角色升级为商家（`role=2`）
