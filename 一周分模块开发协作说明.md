# 一周分模块开发协作说明

项目名称：基于 SpringBoot 和 Vue 的本地特色美食论坛  
适用身份：你负责主导开发，我负责按模块配合你逐步实现  
目标周期：7 天完成可演示版本  

## 1. 这份文档怎么用

你现在不需要一次把整个项目都扔给我开发。我们采用“按模块、按天、按阶段推进”的方式，一次只处理一个小范围任务，这样更稳，也更适合你一周内完成毕设项目。

后面你每次只要做两件事：

1. 告诉我你当前在做哪一天、哪一个模块。
2. 把对应模块的代码、页面、类文件、接口文件或者报错内容发给我。

然后我会根据这个文档，直接继续帮你：

- 拆当前模块任务
- 补代码
- 调整 UI
- 修接口
- 查 bug
- 帮你把当前模块做完整

## 2. 我们的协作方式

### 2.1 你每次发我的内容

你不用每次解释很多，只要按下面格式给我就够了：

#### 如果你在做后端模块

- 当前模块名称
- 涉及的表或字段
- 涉及的 `controller/service/entity/mapper`
- 你现在已经写到哪一步
- 当前报错或卡点

你可以直接这样发：

`现在开发第 2 天，做登录注册模块，这是 UserController、UserService、user 表设计，帮我继续把注册登录和鉴权补完整。`

#### 如果你在做前端模块

- 当前页面名称
- 相关 Vue 页面和组件文件
- 接口地址或 mock 数据
- 你希望达到的视觉效果
- 当前卡点

你可以直接这样发：

`现在开发第 4 天，做首页和列表页，这是首页代码和接口，帮我把 UI 提升到毕设答辩可展示水平，并补齐交互。`

#### 如果你在联调阶段

- 前端请求代码
- 后端接口代码
- 返回数据
- 具体报错截图或文字

### 2.2 我收到后会做什么

我会尽量按下面顺序帮你处理：

1. 先看你当前模块的上下文
2. 明确缺什么
3. 直接补代码或改代码
4. 告诉你下一步该继续发什么

也就是说，你不用自己重新规划流程，我会接着你当前这一步往下推。

## 3. 一周开发总节奏

为了保证一周内能完成“可演示、可答辩”的版本，建议我们按下面 7 天推进。

## 4. 第 1 天：项目骨架和数据库基础

### 4.1 你当天要做什么

- 创建前后端基础工程
- 建立数据库
- 确定项目目录结构
- 配置基础依赖
- 跑通前后端最小可运行版本

### 4.2 当天目标

做到“项目能启动，数据库能连，页面能打开”。

### 4.3 你应该优先完成的内容

#### 后端

- 创建 SpringBoot 项目
- 配置数据库连接
- 配置跨域
- 统一返回结构
- 统一异常处理
- 建好基础包结构

#### 前端

- 创建 Vue 项目
- 配置路由
- 配置请求封装
- 配置基础布局
- 建立全局样式和主题变量

#### 数据库

- 建 `user`
- 建 `food_category`
- 建 `food_post`
- 建 `food_image`
- 建 `comment`
- 建 `like_record`
- 建 `merchant_application`
- 建 `strategy`
- 建 `audit_record`

### 4.4 这一天你要发给我的内容

你可以把下面任一部分发给我：

- SpringBoot 项目结构
- `application.yml`
- 数据库建表 SQL
- Vue 项目结构
- 基础路由文件
- 当前报错

### 4.5 我会帮你做什么

- 帮你检查项目结构是否适合后续扩展
- 帮你修数据库连接、跨域、接口基础配置
- 帮你把建表 SQL 补完整
- 帮你把前端基础布局和主题样式搭好

## 5. 第 2 天：用户模块和登录注册

### 5.1 你当天要做什么

- 完成用户注册
- 完成用户登录
- 完成基础鉴权
- 完成个人资料查询和修改

### 5.2 当天目标

做到“用户可以注册、登录、进入系统、查看和修改自己的资料”。

### 5.3 具体开发内容

#### 后端

- 用户实体类
- 注册接口
- 登录接口
- 密码加密
- 登录态校验
- 获取当前用户资料
- 修改个人信息

#### 前端

- 登录页
- 注册页
- 个人中心基础页
- 登录态保存
- 路由拦截

### 5.4 UI 重点

- 登录页要简洁高级，不要像默认后台模板
- 注册页输入框、按钮、错误提示统一样式
- 个人中心先做基础卡片布局，为后面“我的发布/我的评论”留结构

### 5.5 这一天你要发给我的内容

- `UserController`
- `UserService`
- 登录注册接口代码
- 登录页/注册页 Vue 文件
- 个人中心页面文件

### 5.6 我会帮你做什么

- 把注册登录逻辑补完整
- 修复鉴权和参数校验问题
- 优化登录注册页 UI
- 帮你接上前后端联调

## 6. 第 3 天：分类、美食发布和图片上传

### 6.1 你当天要做什么

- 完成分类管理基础能力
- 完成美食发布功能
- 完成图片上传

### 6.2 当天目标

做到“用户能发布一条带图片、分类、描述的美食内容”。

### 6.3 具体开发内容

#### 后端

- 分类查询接口
- 分类后台管理接口
- 美食发布接口
- 图片上传接口

#### 前端

- 发布页表单

- 图片上传预览

### 6.4 UI 重点

- 上传区域要好看，不能只是一条普通 input
- 表单错误提示明确，避免用户不知道哪里填错

### 6.5 这一天你要发给我的内容

- 发布接口代码

### 6.6 我会帮你做什么

- 当天代码已经提交到本地

- 当天 bug 有记录
- 第二天知道先做什么

## 12. 你每次找我时，最好这样说

为了让我们协作更快，你可以直接复制下面模板来发我。

### 12.1 后端模块模板

`现在是第 X 天，我在做【模块名】。涉及文件有【文件名】。现在已完成【内容】。当前问题是【报错/卡点】。请你直接继续帮我补完整。`

### 12.2 前端模块模板

`现在是第 X 天，我在做【页面名】。涉及文件有【Vue 文件/接口文件】。我想达到【页面效果】。当前问题是【样式/交互/联调】。请你直接帮我改。`

### 12.3 联调模板

`现在是第 X 天，我在联调【模块名】。前端请求代码是【内容】。后端接口代码是【内容】。现在报错是【内容】。请你帮我定位并修。`

## 13. 我们接下来最推荐的实际推进方式

如果你真的准备一周内做完，最稳的方式不是“先想完全部再做”，而是下面这个节奏：

1. 第一天把项目骨架和数据库打牢
2. 第二到第六天每天只攻一个主模块
3. 第七天只做修复、统一和答辩准备

这样你不会乱，我也能每次准确接住你。

## 14. 结论

从现在开始，你不需要再自己想“下一步该做什么”。你只要按这份文档一天一天推进，然后把当前模块发给我，我就会继续帮你把这一块做下去。

你可以把我理解成：

- 你负责主线推进
- 我负责跟着你当前阶段补代码、修 bug、补 UI、补细节

这样最适合你一周冲完一个可答辩版本。
